    DDI Health Checks — Snapshot diario de salud de cada miembro del Grid Infoblox: estado de servicios DNS/DHCP, métricas de leases e IPAM, backup y recursos de hardware (CPU, RAM, Disco).

    ¿Qué información veo aquí?

    • Grid Status Estado global del miembro dentro del Grid.
    • DNS / DHCP Service Estado del servicio en ese miembro (WORKING, FAILED, etc.).
    • DHCP Failover Estado del failover DHCP entre pares.
    • Networks en Riesgo / Críticas Redes con utilización ≥ 80% / ≥ 90%.
    • CPU / RAM / Disco Uso de hardware obtenido via SNMP v3.
    Los datos son publicados por el playbook Ansible DDI_Healthcheck una vez al día vía API REST. Un Grid Status diferente a "WORKING" indica degradación del servicio.
